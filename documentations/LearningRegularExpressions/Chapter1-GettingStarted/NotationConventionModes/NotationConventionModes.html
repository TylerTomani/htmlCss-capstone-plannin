Notation convention & modes

To define a regular expression we place it between two forward slashes

/abc/

these are called "delimiters" which lets us know we are talking about a regular expression

for most of the time we do not need to use these forward slashes but some programing languages require them

flags

there are flags that indicate different modes for regular expressions

  • Standard: /re/ finds only the first match of the expression within the document
  • Global: /re/g - matches expression througout the entire document
  • Case insensitive: /re/i - upper case and lower case is disregarded
  • multiline: /re/m - by default expression cannot span to multiple lines, but m flag allows it
  • unicode: - u
  • sticky - y

grep

flags used to be placed at the beggining

g/re/p

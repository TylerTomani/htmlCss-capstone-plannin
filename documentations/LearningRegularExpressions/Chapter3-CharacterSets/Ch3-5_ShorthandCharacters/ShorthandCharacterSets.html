Shorthand Character Sets

Shorthand

Shorthand Meaning Equivalent
\d Digit [0-9]
\w Word Character [a-zA-Z0-9_]
\s Whitespace [ \t\r\n]
\D Non Digit [^0-9]
\W Non Word Character [^a-zA-z0-9_]
\S Non Whitespace [^ \t\r\n]

These are very useful but they are not in all regular expression engines (especilly old regular expression tools in unix)

caution

Remember, an underscore _ is considered a word character so be careful using the \w shorthand

/\w\w\w/ matches "ABC", "123", "1_A"

while a hypen/dash - is not considered a word character

use /\w\-/ to include the hyphen, this is equivalent to /[a-zA-z0-9_\-/

Examples

example 1

/\d\d\d\d/ matches "1984" but not "text"


example 2

/\w\s\w\w/ matches "I am" but not "Am I"

/\d\s/ while /^\d\s/ will give us the characters

caution

/^\d\s/ is not the same as /\D\S/

/^\d\s/ = NOT a digit OR space character

/\D\S/ = NOT a digit or NOT space, which is alot of things
